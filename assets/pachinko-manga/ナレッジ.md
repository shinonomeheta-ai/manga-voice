# プロジェクトナレッジ（引き継ぎ用コンテキスト）

> このファイルは、Claude Code のパチンコ漫画制作フローに「これまでの文脈と決定事項」を
> 渡すためのものです。新しいセッション・新しい協力者・将来の自分が、ゼロから状況を
> 把握できるように書いてあります。`CLAUDE.md` からも参照されています。

---

## 1. プロジェクトの目的

パチンコ業界を題材にした**ドキュメンタリー漫画**の、シナリオ（ネーム指示書）を
Claude Code で**量産**する。1本ずつ手で書くのではなく、品質を保ったまま
仕組みで回せる状態を作ることがゴール。

---

## 2. 大前提（最初の重要な気づき）

「Claude Code の方が精度が高いのでは？」という出発点があったが、ここは整理済み。

- Claude Code もチャット版の Claude も**中身のモデルは同じ**。よって
  「シナリオ1本の文章としての精度・クオリティ」はどちらで作っても基本変わらない。
- Claude Code の真価は**自動化・量産・仕組み化**にある。具体的には、
  毎回同じテンプレートと設定を読み込ませてブレずに量産する／資料フォルダを
  まとめて参照させる／バッチで複数話を一気に生成する、といった運用。
- したがって本プロジェクトでは、「精度」は単発の賢さではなく
  **仕組み（CLAUDE.md ＋ skill ＋ 蓄積した知識）**で底上げする、という方針を採る。

---

## 3. 採用した構成と技術判断

### なぜこの構成か
Claude Code は起動時に `CLAUDE.md` を自動で読む。これを「作品の憲法」として
共通文脈にし、繰り返す作業は呼び出し可能な手順（skill）に切り出す、という二層構成。
役割ごとにフォルダを分けることで、設定・ネタ元・出力が混ざらないようにしている。

### フォルダ地図
- `CLAUDE.md` … 起動時に自動で読まれる共通文脈・鉄則・地図。
- `ナレッジ.md`（本ファイル）… 文脈と決定事項の引き継ぎ。
- `.claude/skills/scenario/SKILL.md` … 生成手順本体。`/scenario` で起動。
- `設定/作品コンセプト.md` … トーン・ターゲット・連載仕様（**要記入のツマミ**）。
- `設定/パチンコ知識メモ.md` … 用語と事実の土台（**育てる前提**）。
- `資料/` … 各話のネタ元。1ファイル＝1題材。
- `テンプレート/シナリオ出力形式.md` … 出力の統一フォーマット。
- `出力/` … 完成シナリオの保存先（`第NN話_タイトル.md`）。

### 形式についての技術判断（裏取り済み）
- カスタムコマンドの推奨形式は `.claude/skills/<名前>/SKILL.md`。
  `/名前` で手動起動でき、Claude が自動でも使える。
- 旧来の `.claude/commands/<名前>.md` は**レガシー扱い**で、現在もCLIで動くが
  非推奨。2026年4月のアップデートでコマンドはスキルへ統合された。
- 引数は `$ARGUMENTS`（全引数）や `$0` `$1`（位置引数）で受け取れる。
- SKILL.md の本文は**使うときだけ読み込まれる**ため、長い手順を入れても
  普段のコンテキストを圧迫しない。
- 出典: code.claude.com の slash-commands / agent-sdk ドキュメント、および
  Claude Code 公式ドキュメント。仕様は更新が速いので、迷ったら最新版を確認する。

## 4. 制作の決め事（編集方針＝全シナリオ共通の鉄則）

1. **事実が主役**。脚色は読者の理解を助ける範囲に留め、創作した部分は
   出力末尾の「監修メモ」に必ず明記する。
2. **ギャンブルを煽らない**。必勝法や射幸心を直接あおる描写はしない。
   依存・家計・規制など、ドキュメンタリーとしての多面性を必ず保つ。
3. 実在の個人・店舗・メーカーを断定的に悪く描かない。必要なら仮名・モデル化。
4. **1話完結**を基本としつつ、語り手を固定して**連載**として積み上げる。
5. 出力は必ず `テンプレート/シナリオ出力形式.md` の形式に従う。
6. 年号・数字・市場規模などは印象で書かず、**資料で裏を取る**。
   未確認の点は監修メモに「要・事実確認」として残す。

---

## 5. 生成フロー

新しい話を作るときの基本動作:

    /scenario 資料/（題材ファイル）.md      # 既存メモから
    /scenario 1990年代の換金問題             # テーマ直指定

`/scenario` の内部手順は「設定を読む → 資料/テーマを確定 → ドキュメンタリーとして
設計 → テンプレ形式で出力 → `出力/` に保存 → 報告」の固定フロー。

量産（バッチ）したいときは起動後にこう頼む:

> 資料/ にある全ファイルから1話ずつシナリオを作って、出力/ に保存して

---

## 6. 精度を上げ続ける運用ルール

この仕組みは「育てるほど賢くなる」設計。回すたびに次を意識する。

- **`設定/パチンコ知識メモ.md` に事実を貯める**。取材で得た年号・用語・
  エピソードを書き足すほど、生成シナリオの事実精度が上がる。
- **`設定/作品コンセプト.md` を具体化する**。トーンや語り手が曖昧だと出力もぶれる。
- 良かった出力／ダメだった出力の傾向に気づいたら、鉄落（鉄則）や
  テンプレート、SKILL.md の手順に反映していく。

## 8. 一言サマリ

「単発の賢さ」ではなく「仕組みと蓄積」で量産品質を担保するプロジェクト。
鉄則はドキュメンタリーとしての誠実さ（事実が主役・煽らない・裏取り）。
迷ったら CLAUDE.md の鉄則と、このファイルの方針に立ち返る。
